# Como esta a interface do projeto

Este arquivo explica, de forma simples, como a interface do Projeto Agenda esta hoje.

## Entrada do sistema

Quando a aplicacao abre, a pessoa encontra primeiro a tela de login.

Essa tela tem:

- campo de email
- campo de senha
- botao para entrar
- link para ir para o cadastro

## Tela de cadastro

Se a pessoa ainda nao tiver conta, pode abrir a tela de cadastro.

Nela existem campos para:

- nome
- email
- senha
- confirmacao de senha
- telefone
- CPF
- data de nascimento

Depois do cadastro, o sistema leva a pessoa de volta para o login.

## Area autenticada

Depois que o usuario entra, ele vai para uma area autenticada chamada dashboard.

Hoje essa area ainda e simples. Ela mostra:

- um titulo de dashboard
- uma mensagem inicial
- um menu lateral

## Menu lateral

O menu lateral permite navegar para:

- dashboard
- agenda
- perfil
- sair da conta

No celular ou em telas menores, esse menu pode ser aberto por um botao.

## Como a interface parece hoje

Visualmente, a interface atual segue um estilo escuro, com:

- fundo em degradiente escuro
- cartoes centrais para login e cadastro
- botoes com destaque visual
- animacao simples de entrada

## O que ainda nao esta pronto na interface

Apesar de a navegacao existir, algumas paginas ainda nao tem conteudo montado.

Hoje:

- a pagina de agenda ainda esta vazia
- a pagina de perfil ainda esta vazia
- ainda nao existem telas prontas para mostrar tarefas e eventos

## Resumindo

A interface do projeto ja permite entrar no sistema, criar conta e acessar uma area autenticada inicial.

O proximo passo da interface e preencher as paginas de agenda e perfil com as funcoes que o backend ja comecou a oferecer.
